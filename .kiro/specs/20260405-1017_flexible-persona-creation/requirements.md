# Requirements: データ＋プロンプトによるペルソナ作成の柔軟性強化

## Background & Context
### User Problems
- 現状のペルソナ生成UIは「インタビューデータ」「調査レポート」の2タブに分かれており、データ種別ごとに固定のロジックで生成される
- レビューデータや購買データなど、多様なデータからペルソナを作成したいニーズに対応できない
- ユーザーが「どのようなペルソナを作りたいか」を指示する手段がない

### Related Issues
- GitHub Issue #9: データ＋プロンプトによるペルソナ作成の柔軟性強化

## Objectives
- あらゆるデータファイルから、自然言語の指示でペルソナを作成できる統一UIを提供する
- 既存の生成ロジック（インタビュー/レポート）はプリセットとして後方互換を維持する

## Scope
### In Scope (Phase 1)
- 統一ペルソナ生成UIの実装（既存generation.htmlの置き換え）
- ファイルアップロード（複数可、1データソースのみ）
- データ種別選択（プリセット: インタビューデータ / 市場調査レポート / レビューデータ / 購買データ / その他）
- カスタムプロンプト入力（任意、ペルソナ作成の追加指示）
- 生成数指定（1〜10）
- データ種別＋カスタムプロンプトからプロンプトを構築するペルソナ作成エージェント
- 結果確認・選択・保存

### Out of Scope (Phase 2以降)
- マスサーベイデータソース連携（Nemotron / カスタムCSV）
- 外部データセット連携
- 複数データソースの組み合わせ
- プリセットのユーザー追加・管理機能

## Detailed Requirements

### R1: 統一UIフロー
ユーザーは以下のステップでペルソナを生成する:
1. ファイルアップロード（複数可、対応形式: .txt, .md, .pdf, .docx, .doc, .csv）
2. データ種別選択（プリセットまたは「その他」でフリーテキスト入力）
3. カスタムプロンプト入力（任意）
4. 生成数指定（1〜10、デフォルト: 1）
5. 生成実行
6. 結果確認・選択・保存

### R2: データ種別プリセット
以下のプリセットを提供する:
- **インタビューデータ**: N1インタビュー、顧客ヒアリング等
- **市場調査レポート**: 市場分析、業界レポート等
- **レビューデータ**: 商品レビュー、口コミ等
- **購買データ**: 購買履歴、トランザクションデータ等
- **その他**: フリーテキストでデータの説明を入力

### R3: カスタムプロンプト
- 任意入力フィールド
- ペルソナ作成の追加指示を自然言語で記述
- 例: 「30代女性で製品に不満を持つペルソナを作って」

### R4: ペルソナ作成エージェント
- データ種別の説明＋カスタムプロンプト＋アップロードデータからプロンプトを動的構築
- Strands Agent SDKベースで実装
- 1〜10個のペルソナを生成可能

### R5: 後方互換性
- 既存の「インタビューデータからペルソナ生成」はプリセット選択で同等の結果を得られる
- 既存の「レポートからペルソナ生成」もプリセット選択で同等の結果を得られる
